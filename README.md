The engine is built using a structured class system that handles character stats, enemy scaling, and a custom rendering pipeline. I designed it to be highly interactive, moving away from standard turn-based combat by introducing reflex-based mechanics. For example, the Fallen Angel class is built around a system of "divine punishment" where the player has to type specific words under a time limit to perform actions or escape traps. This required careful timing logic using Python's time module and a way to refresh the terminal screen without flickering, which I achieved by building a custom UI buffer.

I spent a lot of time on the "Fallen Angel" mechanics because they represent a different kind of difficulty. Instead of just making enemies hit harder, I implemented curses that physically restrict the player, such as manacles that force typing challenges or sensory deprivation that hides the user interface entirely. The combat loop also includes a difficulty scaling system that adjusts enemy health and damage based on the party's level, along with a shop system where players can manage their gold to buy essential survival items. The visual side uses ANSI escape codes to handle colors and screen-shake effects, giving the terminal a more dynamic and "game-like" feel.

This project taught me a lot about object-oriented programming and the importance of organizing data. Managing so many different status effects and character classes required a clean stat-update matrix and an efficient way to render ASCII art based on the player's current state. It also pushed me to think about user experience in a text-based environment, specifically how to use dramatic prompts and color-coding to convey danger and urgency. In the future, I might look into adding a save/load feature using JSON or expanding the local co-op features to support more complex party interactions.
